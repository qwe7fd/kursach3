Функциональные требования к приложению "Справочник целебных трав"

1.  Запуск приложения: Главное окно приложения должно открываться по центру экрана.
2.  Выход из приложения: При нажатии на кнопку "Выход" в главном меню приложение должно полностью завершать свою работу (закрываться). (**ОШИБКА**: Окно скрывается, но процесс остается в памяти)
3.  Главное окно: Окно не должно изменять свои размеры (фиксированные границы), чтобы не ломать верстку. (**ОШИБКА**: Окно можно растягивать, ломая интерфейс)
4.  Интерфейс: Основной цвет фона всех форм приложения должен быть белым. (**ОШИБКА**: Фон стандартный серый (Control), а не белый)
5.  Добавление растения (Валидация): Поле "Название растения" является обязательным для заполнения.
6.  Добавление растения (Валидация): Поле "Лечимые заболевания" является обязательным для заполнения. (**ОШИБКА**: Проверка этого поля закомментирована, можно сохранить растение без заболеваний)
7.  Добавление растения (Данные): Данные о новом растении должны сохраняться в файл "herbs.txt".
8.  Файловая система: Если файл herbs.txt отсутствует, приложение не должно выдавать ошибку, а должно считать, что список растений пуст. (**ОШИБКА**: При отсутствии файла появляется MessageBox с ошибкой)
9.  Добавление растения (Формат): В файле хранения растений поля должны быть разделены точкой с запятой (\;).
10. Добавление растения (Уникальность): Система не должна позволять добавлять растение с названием, которое уже существует в базе. (**ОШИБКА**: Проверка на уникальность отсутствует, можно создавать дубликаты)
11. Добавление растения (Сообщение): При успешном сохранении должно выводиться сообщение, содержащее название добавленного растения. (**ОШИБКА**: Выводится общее сообщение "Растение успешно добавлено!" без названия)
11. Добавление растения (Навигация): Кнопка "Назад" должна закрывать форму добавления растения.
12. Добавление растения (Поля): Система должна сохранять описание внешнего вида растения.
13. Добавление растения (Интерфейс): Плейсхолдер для поля "Место произрастания" должен быть "Место произрастания" (без опечаток). (**ОШИБКА**: В плейсхолдере опечатка: "Место произрастния")
14. Добавление рецепта (Заголовок): Заголовок окна добавления рецепта должен быть "Новый рецепт". (**ОШИБКА**: Заголовок окна "Добавление рецепта")
14. Добавление рецепта (Валидация): Поле "Название недуга" является обязательным для заполнения.
15. Добавление рецепта (Валидация): Поле "Инструкция" является обязательным для заполнения. (**ОШИБКА**: Из-за ошибки copy-paste вместо поля "Инструкция" повторно проверяется поле "Название недуга")
16. Добавление рецепта (Ингредиенты): Рецепт должен содержать минимум один выбранный ингредиент.
17. Добавление рецепта (Список): Список доступных ингредиентов должен формироваться на основе сохраненных растений.
18. Добавление рецепта (Сохранение): Данные о рецептах должны сохраняться в файл "recipes.txt" в кодировке UTF-8.
19. Добавление рецепта (Поведение): После успешного сохранения рецепта форма должна автоматически закрываться. (**ОШИБКА**: Форма не закрывается, а только очищает поля)
20. Добавление рецепта (Навигация): Кнопка "Назад" должна закрывать форму создания рецепта. (**ОШИБКА**: Кнопка "Назад" не выполняет никаких действий)
21. Удаление рецепта (Список): Список рецептов в окне удаления должен быть отсортирован по алфавиту (по названию недуга). (**ОШИБКА**: Список не отсортирован и отображается в порядке записей в файле)
22. Удаление рецепта (Детали): При выборе рецепта в списке должна отображаться его инструкция и список ингредиентов.
23. Удаление рецепта (Подтверждение): Перед удалением рецепта должно появляться диалоговое окно с просьбой подтвердить действие, и удаление должно происходить только при нажатии "Да". (**ОШИБКА**: Код удаления выполняется независимо от выбора пользователя в диалоговом окне)
24. Удаление рецепта (Навигация): Кнопка "Назад" должна закрывать окно удаления рецепта.
25. Удаление рецепта (Поиск): Поле поиска в окне удаления рецепта должно фильтровать список по названию недуга.
26. Поиск растения (Имя): Поиск растения по названию не должен зависеть от регистра вводимых символов (строчные/заглавные).
27. Поиск растения (Болезнь): Поиск растения по лечимому заболеванию не должен зависеть от регистра символов.
28. Поиск (Частичное совпадение): Поиск должен находить элементы, если введенная строка является частью названия (например, "рома" находит "Ромашка"). (**ОШИБКА**: Поиск работает только по началу строки (StartsWith), а не по вхождению (Contains))
29. Отображение данных: В детальном просмотре рецепта ингредиенты должны перечисляться через запятую.
30. Файловая система: Приложение должно корректно работать с файлами в кодировке UTF-8.
